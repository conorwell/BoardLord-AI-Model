# Local Data Representation

## Directory Structure

```
board_lord.ai/
├── sync_files/            # Raw API responses (fetch_001.json, fetch_002.json, ...)
├── holes.json             # All 1177 holes from the API (fetched once)
├── placements.json        # All 1299 placements from the API (fetched once)
├── position_map.json      # position_id → (x, y) for TB2 Mirror (layout 10)
├── position_map_spray.json# position_id → (x, y) for TB2 Spray (layout 11)
├── mirror_map.json        # position_id → mirrored_position_id for TB2 Mirror

```

## holes.json

Fetched from the API with `holes=1970-01-01+00%3A00%3A00.000000`. Contains 1177 records.

```json
{
  "id": 123,
  "product_id": 1,
  "name": "A5",
  "x": -32,
  "y": 68,
  "mirrored_hole_id": 456,
  "mirror_group": 12
}
```

- `x`: horizontal position in inches from center. Range: -64 to 64, step 4.
- `y`: vertical position in inches from floor. Range: 4 to 140, step 4. **y increases upward.**
- `mirrored_hole_id`: the ID of the symmetric hole on the other side of the board.

## placements.json

Fetched from the API with `placements=1970-01-01+00%3A00%3A00.000000`. Contains 1299 records.

```json
{
  "id": 400,
  "layout_id": 10,
  "hole_id": 88,
  "set_id": 12,
  "default_placement_role_id": 12
}
```

- `id`: the **placement_id**, which is what `frames` strings in climb data refer to.
- `layout_id`: which board layout this placement belongs to.
- `hole_id`: foreign key into `holes.json` — use this to get (x, y).
- `set_id`: 12 = wood holds, 13 = plastic holds. **Used directly by the model as a grid channel.**

### Layout ranges

| Layout | Name | Placement ID range | Hold types |
|--------|------|--------------------|------------|
| 10 | TB2 Mirror | 304–801 (498 placements) | 304–545: wood, 546–801: plastic |
| 11 | TB2 Spray | 802–1299 (498 placements) | 802–1043: wood, 1044–1299: plastic |

## mirror_map.json

Pre-computed lookup: `position_id (str) → mirrored_position_id (int)` for TB2 Mirror.

Built by `build_mirror_map.py` by chaining:
```
position_id → hole_id (placements.json)
hole_id → mirrored_hole_id (holes.json)
mirrored_hole_id → position_id (reverse lookup in placements.json)
```

All 498 TB2 Mirror positions have a mirror. Used by `expand_annotations.py` to halve hold annotation effort — annotate one side, auto-fill the other.

---

## position_map.json / position_map_spray.json

Pre-computed lookup: `position_id (str) → [x, y]`.

Built by joining `placements` → `holes` for a given layout_id. Avoids the join at runtime.

```json
{
  "304": [-64, 4],
  "305": [-56, 4],
  ...
  "801": [64, 140]
}
```

### Grid mapping (used in model encoding)

The board coordinate system maps to a 35×33 integer grid:

```python
col = (x + 64) // 4    # 0–32, left → right
row = (140 - y) // 4   # 0–34, top → bottom (row 0 = top of board)
```

So the model grid is oriented the same way you'd see the board hanging on a wall.

### Corner positions (TB2 Mirror)

| Placement ID | Corner | x | y |
|-------------|--------|---|---|
| 679 | Top Left | -64 | 140 |
| 672 | Bottom Left | -64 | 4 |
| 794 | Bottom Right | 64 | 4 |
| 801 | Top Right | 64 | 140 |

## hold_labels.json

Manual per-hold annotations keyed by position_id. See `docs/annotation.md` for the full annotation guide and workflow.

```json
{
  "304": { "incut": 20,  "depth": 1.5, "pinchability": 0.1 },
  "305": { "incut": -5,  "depth": 2.5, "pinchability": 0.0 }
}
```

| Field | Range | Description |
|-------|-------|-------------|
| `incut` | ~-30 to 90° | Edge angle. 0 = flat, negative = sloping, 50 = incut crimp |
| `depth` | 0.5–3.0 pads | How much finger fits. 1 = pad crimp, 3 = full finger/jug |
| `pinchability` | 0–1 | Thumb engagement. 0 = no pinch, 1 = full symmetric pinch |

Unannotated positions default to `incut=10, depth=1.5, pinchability=0.1`.

## Parsed Climb JSON (TB2_Mirror_Problems / TB2_Spray_Problems)

Generated by `parse_climbs.py`. One file per (climb × angle) combination.

```json
{
  "uuid": "5e0b6f2b3cdb46b392e79a81d4d2c59b",
  "name": "007 licencia para decotar",
  "frames": "p338r8p394r5p412r6p801r7",
  "holds": [
    {"position_id": 338, "role": 8},
    {"position_id": 394, "role": 5},
    {"position_id": 412, "role": 6},
    {"position_id": 801, "role": 7}
  ],
  "original_angle": 35,
  "is_nomatch": true,
  "layout_id": 10,
  "angle": 35,
  "difficulty_average": 24.0,
  "quality_average": 3.0,
  "ascensionist_count": 1,
  "benchmark_difficulty": null
}
```

### Key fields

| Field | Description |
|-------|-------------|
| `frames` | Raw encoded string from API |
| `holds` | Parsed list of {position_id, role} dicts |
| `angle` | Board angle this stat record applies to (0–65°, multiples of 5) |
| `original_angle` | The angle the setter originally tagged when creating the problem |
| `difficulty_average` | Continuous difficulty score, ~10–33. 1 unit = 1 French grade. |
| `quality_average` | 1–3 star rating averaged across all raters |
| `ascensionist_count` | Number of logged ascents |
| `benchmark_difficulty` | Community-agreed "classic" difficulty, if designated |
| `is_nomatch` | True if the problem has no matchable hold. False = can match (generally easier). Used as a scalar model input. |

**Note:** A single climb UUID may appear as multiple files if it has stats at multiple angles (e.g. `MyClimb_25.json`, `MyClimb_40.json`).

## Difficulty Scale

`difficulty_average` is a continuous float, roughly 10–33:

| difficulty_average | French Grade | V-Grade | Model Class |
|-------------------|--------------|---------|-------------|
| 10–12 | 4A–4C | V0 | 0 (≤V1) |
| 13–14 | 5A–5B | V1 | 0 (≤V1) |
| 15 | 5C | V2 | 1 |
| 16–17 | 6A–6A+ | V3 | 2 |
| 18–19 | 6B–6B+ | V4 | 3 |
| 20–21 | 6C–6C+ | V5 | 4 |
| 22 | 7A | V6 | 5 |
| 23 | 7A+ | V7 | 6 |
| 24–25 | 7B–7B+ | V8 | 7 |
| 26 | 7C | V9 | 8 |
| 27 | 7C+ | V10 | 9 |
| 28+ | 8A–8B | V11–V13 | 10 (V11+) |

**Grade collapsing rationale:**
- **≤V1 (class 0)**: V0 and V1 are genuinely hard to distinguish on a tension board — problems at this level have many holds and high beta variance. Collapsed to reduce label noise at the easy end.
- **V11+ (class 10)**: V11/V12/V13 have very few ascents (131/63/32 raw) and community consensus on these grades is unclear even among elite climbers. Collapsed to avoid training on unreliable labels with tiny sample sizes.

The scale is **not linear in physical difficulty** — the gap between V0 and V1 is much smaller than the gap between V10 and V11.

## Hold Roles

| Role ID | Name | Meaning |
|---------|------|---------|
| 5 | Start | Must be used to begin the problem |
| 6 | Hand | Regular hand hold |
| 7 | Finish | Must be matched or controlled to complete |
| 8 | Foot | Foot-only — cannot be used as a handhold |

Feet are **never open** on the Tension Board. Any hold not listed in `frames` is not in use.

## Data Quality Filters (used in model training)

Problems are filtered before training:

- `difficulty_average` must be non-null
- `quality_average` ≥ 1.5
- Ascent count threshold varies by grade (hard problems get fewer ascents by nature):
  - 7C+ and above (diff ≥ 27): 1 ascent minimum
  - 7A+ through 7C (diff ≥ 23): 2 ascents minimum
  - All others: 2 ascents minimum

After filtering, TB2 Mirror yields approximately 9000 training samples (18000 after horizontal flip augmentation).
